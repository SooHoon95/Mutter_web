# 실기기 디바이스 검증 체크리스트 (T10 US-010 AC-3)

**대상:** iOS Safari (iPhone) · Android Chrome  
**방법:** Netlify 미리보기 배포 URL로 실기기에서 수동 검증  
**통과 기준:** 각 항목의 "기대 결과"가 실제로 관찰되어야 한다.

---

## 공통 준비

- [ ] Netlify 미리보기 배포 URL 확보 (예: `https://deploy-preview-XXX--letter-app.netlify.app`)
- [ ] 테스트용 발신자 계정 + 매직링크 로그인 완료 확인
- [ ] 테스트용 편지 1통 작성 (SC 큐 1단락 + CC0 큐 1단락) 및 링크 발급
- [ ] 링크 URL을 별도 메모 또는 QR코드로 준비 (인코그니토 테스트용)

---

## 1. iOS Safari (iPhone)

### 1-1. 콜드 로드 성능

| 항목 | 기대 결과 | 결과 |
|------|----------|------|
| 앱 첫 로드 (4G / LTE 환경) | 인터랙티브까지 **3초 이내** | ☐ |
| JS 번들 로드 후 첫 화면(랜딩) 표시 | "연출되는 편지" 헤딩 표시 | ☐ |

### 1-2. PWA / A2HS (홈 화면 추가)

| 항목 | 기대 결과 | 결과 |
|------|----------|------|
| Safari 공유 시트에서 "홈 화면에 추가" 메뉴 표시 | 메뉴가 노출됨 | ☐ |
| 홈 화면 아이콘 추가 후 독립 실행 (standalone) | Safari 주소바 없이 앱 셸로 실행 | ☐ |
| 홈 화면 아이콘 이름 | "편지" | ☐ |
| theme_color 상태바 | 어두운 배경 (#1a1a1a) | ☐ |

### 1-3. "편지 열기 ▶" 단일 제스처 오디오 언락

| 항목 | 기대 결과 | 결과 |
|------|----------|------|
| 인코그니토 탭으로 `/l/:token` 오픈 | "도착한 편지가 있어요." 게이트 표시 | ☐ |
| "편지 열기 ▶" 탭 **1회** | 게이트 사라지고 편지 본문 표시 | ☐ |
| 추가 탭 없이 음악 시작 | CC0 or SC 오디오가 자동 시작됨 (추가 탭 불필요) | ☐ |
| iOS 오디오 언락 확인 | 오디오 팝업 없이 재생 시작 | ☐ |

### 1-4. 스크롤 동기 음악

| 항목 | 기대 결과 | 결과 |
|------|----------|------|
| 스크롤 내리다가 SC 큐 단락 진입 | SC 음악 (또는 CC0 폴백)이 재생/전환 | ☐ |
| 단락 이탈 후 다음 단락 진입 | 음악 seekTo 또는 트랙 전환 | ☐ |
| 스크롤 도중 끊김 없이 재생 | 버퍼링·침묵 없음 | ☐ |

### 1-5. SC 폴백 (CC0 무음0 보장)

| 항목 | 기대 결과 | 결과 |
|------|----------|------|
| SC 큐가 있는 편지에서 SC 로드 실패 시뮬레이션 (네트워크 차단) | CC0 카탈로그 트랙으로 자동 대체 재생 | ☐ |
| 본문이 사라지지 않음 | 편지 텍스트 계속 표시 | ☐ |
| 에러 메시지 없음 | "열 수 없어요" 등 에러 화면 미표시 | ☐ |

### 1-6. 레이아웃 / 접근성

| 항목 | 기대 결과 | 결과 |
|------|----------|------|
| 가로 스크롤 없음 | 수평 스크롤바 미표시, 콘텐츠 잘림 없음 | ☐ |
| Safe area (노치/Dynamic Island) | 콘텐츠가 노치에 가려지지 않음 | ☐ |
| 폰트 크기 적절 | 최소 16px 이상, 본문 읽기 편함 | ☐ |

### 1-7. noindex

| 항목 | 기대 결과 | 결과 |
|------|----------|------|
| `/l/:token` 페이지 소스 확인 | `<meta name="robots" content="noindex">` 존재 | ☐ |
