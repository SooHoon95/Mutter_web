# verify-task

TASK의 수용기준(AC)을 **신선한 증거**와 함께 검증한다(ralph 스타일). 자가 승인 금지 — 별도 검증 패스.

## 입력
- 대상 TASK ID(US-00x)와 `.omc/prd.json`의 해당 acceptanceCriteria.

## 작업 순서

1. **AC 나열:** `.omc/prd.json`에서 해당 스토리의 모든 acceptanceCriteria를 읽는다.
2. **각 AC마다 신선한 증거 수집:**
   - 해당 동작을 증명하는 테스트/빌드/로그를 실제로 실행하고 출력을 읽는다.
   - "될 것이다/괜찮아 보인다" 금지 — 실행 결과로 입증.
   - 증거 예: `npm run test`(특정 spec) 통과 출력, `npm run build` 성공, e2e 통과, 수동 확인 로그.
3. **미충족 AC가 하나라도 있으면:** 완료로 표시하지 않는다. 원인 진단 → 수정 → 재검증.
4. **별도 리뷰 패스:** 변경 규모에 맞게 `code-reviewer`(표준) 또는 `architect`(아키텍처·보안·다중 시스템) 서브에이전트로 AC 대비 교차 검증. 같은 활성 컨텍스트 자가 승인 금지.
5. **통과 시:** `.omc/prd.json`의 해당 스토리 `passes: true` 설정, `tasks/todo.md` 리뷰 섹션에 증거 기록, TaskUpdate로 완료.

## 자가 점검
- 각 AC에 실행된 신선한 증거가 1:1로 매핑되는가.
- 테스트를 삭제/약화해 통과시키지 않았는가.
- 별도 리뷰어 패스가 AC 대비로 승인했는가.
